网络编程和套接字

发表于 2019-08-13 | 分类于 TCP/IP网络编程 | 浏览 次

首先实现一个简单的服务器端和客户端通信的小例子．
服务器端代码

服务器端调用步骤：

  1. 调用socket函数创建套接字
  2. 调用bind函数分配IP地址和端口号
  3. 调用listen函数将套接字转为可接收连接状态
  4. 调用accept函数受理连接请求．如果在没有连接请求的情况下调用该函数，则不会返回，直到有连接请求为止．
  5. 使用write函数用于向客户端传输数据．

  • 创建socket

    socket函数成功时返回一个文件描述符(见相关知识)，失败则返回-1.

  • 描述socket

    • 接下来代码描述socket文件，包括socket文件的大小，接受的内容等

    • AF_INET　IPV4网络协议的套接字类型

    • htonl(INADDR_ANY) 告诉系统，这个socket监听所有的网络，即指定地址为0.0.0.0

    • htons(atoi(argv[1]))告诉系统监听的端口，该端口由服务器程序使用者输入

    • htonl将主机数转换成无符号长整形的网络字节顺序．本函数将一个32位数从主机字节顺序转换成网络字节顺序．

    • htons将整形变量从主机字节顺序转变成网络字节顺序．即整数在地址空间存储方式变为高位字节存放在内存的低地址处．

    • htonl和htons代表了将本地的字符串字节序转化为网络字节序，防止因为本地的大小端存放问题导致出错．

  • （地址）绑定

    • 给套接字分配地址信息（IP地址和端口号）

  • 监听

    • 绑定了地址信息后，只需要最后一步接收来自客户的连接请求了．这就是让套接字变成监听状态，之后，只要客户端向服务器发送请求，服务器就可以接收到了．

  • 接受连接

    • accept函数成功时返回文件描述符，失败时返回-1
    • 一个服务器端socket可以接收很多客户端的连接请求．当客户端发起连接请求时，服务器端需要调用accept函数受理客户端的请求．同时来自不同客户端的连接也会当成不同的socket文件，返回不同的文件描述符．

相关知识

  • 文件描述符
    • 对linux来说，一切皆文件．套接字和本地的文件都是文件．
    • 文件描述符是系统分配给文件或套接字的整数．是为了方便称呼操作系统分配的文件或套接字．
  • 结构体sockaddr_in
